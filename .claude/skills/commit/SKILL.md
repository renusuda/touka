# Git Commit

- Use conventional commits
  - `feat:` — New user-facing functionality
  - `fix:` — Bug fix
  - `refactor:` — Code restructuring with no behavior change
  - `docs:` — Changes to documentation, code comments, docstrings, or READMEs (no logic or behavior changes)
  - `test:` — Adding or updating tests (no production code changes)
  - `chore:` — Build config, dependencies, and other housekeeping (not user-facing)
  - `ci:` — CI/CD configuration changes
- Keep commit messages concise (1-2 sentences)
